[IDK wtf this means:] Array Allocation:  protected abstract PanelType[] ActivePanels { get; }  means every time a derived class returns  new PanelType[] { ... } , it allocates memory. For state transitions that might happen often, it's better to declare a  protected abstract readonly PanelType[] ActivePanels  or ensure the derived class returns a cached array rather than creating a  new  array on every  get. As predicted in the base class review,  protected override PanelType[] ActivePanels => new[] { ... };  creates a new array every single time  ActivePanels  is accessed. Since UIBaseState.OnEnter  and  OnExit  access  ActivePanels  using a foreach loop, it's allocating a new array multiple times when transitioning. You should cache it: private static readonly PanelType[] _activePanels = new[] { PanelType.MainBg, PanelType.CreditsMenu }; protected override PanelType[] ActivePanels => _activePanels; This completely eliminates garbage collection overhead for UI transitions.
